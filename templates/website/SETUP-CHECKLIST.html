Conquer Computers Setup Checklist

CONQUER COMPUTERS LLC - FINAL CPANEL SETUP CHECKLIST
====================================================

This package is updated for cPanel hosting and does NOT require Firebase Storage.
Complaint attachments upload to cPanel through complaint-upload-handler.php.

UPLOAD FIRST
------------
1. Take a backup of your existing public_html.
2. Extract this ZIP.
3. Upload the CONTENTS inside conquercomputers.com to public_html.
4. Confirm these files exist:
   - public_html/index.html
   - public_html/lead-handler.php
   - public_html/job-pdf-email-handler.php
   - public_html/complaint-upload-handler.php
   - public_html/private-config.sample.php
   - public_html/static/js/script.js
   - public_html/static/js/portal.js
   - public_html/sitemap.xml
   - public_html/robots.txt

4. SETUP CPANEL EMAIL PROPERLY
------------------------------
Your lead form sends email to: info@conquercomputers.com

In cPanel:
1. Go to Email Accounts.
2. Create/confirm info@conquercomputers.com.
3. Send a normal test email from Gmail to info@conquercomputers.com.
4. Open cPanel Webmail and confirm the message arrives.
5. Go to Email Deliverability.
6. Confirm SPF and DKIM are valid.

If lead email does not arrive:
- Check Webmail inbox and spam.
- Check cPanel Email Deliverability.
- Check hosting mail limits.
- If PHP mail() is blocked, upgrade later to SMTP/PHPMailer.

5. TEST PUBLIC LEAD FORM
------------------------
Open: https://www.conquercomputers.com
Submit a test lead:
- Name: Test Lead
- Email: your Gmail
- Phone: your UAE number
- Service: IT AMC Services
- Requirement: Testing website lead form

Check:
1. Company email received at info@conquercomputers.com.
2. Customer auto-reply received in the test Gmail.
3. Firestore > leads collection has the new lead.
4. WhatsApp opens with lead details.
5. GA4 Realtime shows lead_form_submit.
6. cPanel creates private-leads/leads-YYYY-MM.csv.

Important: The updated flow sends to PHP/email/local backup first and saves to Firestore second.
So even if Firestore rules fail, the lead can still be saved in cPanel and/or emailed.

6. SETUP GA4 EVENTS AS KEY EVENTS
---------------------------------
Open Google Analytics:
Admin > Data display > Events

Submit one test lead and wait a few minutes.
Mark these events as key events:
- lead_form_submit
- phone_click
- whatsapp_click
- get_quote_click

Most important for ads:
- lead_form_submit
- phone_click
- whatsapp_click

7. CONNECT GA4 WITH GOOGLE ADS
------------------------------
Only do this after GA4 events are visible.

Go to:
GA4 > Admin > Product links > Google Ads Links

Then:
1. Connect Google Ads account.
2. In Google Ads, import GA4 key events.
3. Do not run ads until conversions are imported.

8. SETUP GOOGLE SHEET / CRM LEAD SAVING
---------------------------------------
Create a Google Sheet named: Conquer Computers Leads

Recommended first sheet tab name: Leads

Columns are automatically handled by google-apps-script-leads.js.

Setup:
1. Open the Google Sheet.
2. Go to Extensions > Apps Script.
3. Copy code from google-apps-script-leads.js.
4. Paste into Apps Script.
5. Deploy > New deployment > Web app.
6. Execute as: Me.
7. Who has access: Anyone.
8. Copy the Web App URL.
9. Rename private-config.sample.php to private-config.php in cPanel.
10. Add the URL here:

   define('GOOGLE_SHEET_WEBHOOK_URL', 'PASTE_WEB_APP_URL_HERE');

After this, leads can save to:
- Email
- Firestore
- Local cPanel CSV backup
- Google Sheet

9. SETUP PRIVATE CRM VIEWER
---------------------------
The package includes: lead-crm.php

In private-config.php set:

define('CRM_SECRET_TOKEN', 'CHANGE_TO_A_STRONG_PRIVATE_TOKEN');

Then open:
https://www.conquercomputers.com/lead-crm.php?token=CHANGE_TO_A_STRONG_PRIVATE_TOKEN

This viewer reads cPanel local lead backups from private-leads/*.csv.
Do not share this URL publicly.

10. SETUP PRIVATE BACKUP TOKEN
------------------------------
The package includes: cron-backup.php

In private-config.php set:

define('BACKUP_SECRET_TOKEN', 'CHANGE_TO_A_STRONG_BACKUP_TOKEN');

Manual test:
https://www.conquercomputers.com/cron-backup.php?token=CHANGE_TO_A_STRONG_BACKUP_TOKEN

For daily cPanel cron:
php -q /home/YOUR_CPANEL_USERNAME/public_html/cron-backup.php

Replace YOUR_CPANEL_USERNAME with the real cPanel username.
